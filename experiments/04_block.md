# Transformer Block：Attention と MLP の協力

GPT-2 のモデルは、この「Transformer Block（トランスフォーマー・ブロック）」を 12 層積み重ねることで構成されています。

1 つの Block は、大きく分けて **Attention（文脈の結合）** と **MLP（情報の深掘り）** という 2 つのステージで構成されています。

---

## 1. 構造の全体像：Pre-LayerNorm と 残差接続

GPT-2 の各 Block は、次のような流れでベクトルの意味をアップデートします。

1. **LayerNorm 1**: 入力ベクトルの統計的な偏りをなくす。
2. **Attention**: 周囲の単語から「文脈」を取り込む。
3. **残差接続 1**: 「元のベクトル」に「Attention の差分」を**足す**。
4. **LayerNorm 2**: 再び正規化する。
5. **MLP**: 単語単体の情報を高次元（3072次元）に広げ、深く処理する。
6. **残差接続 2**: 「ここまでのベクトル」に「MLP の差分」を**足す**。

### なぜ「足す（残差接続）」のか？
`x = x + output` という形にすることで、層がどれだけ深くなっても「元の情報」が失われにくくなります。学習の初期段階では「何もしない（恒等写像）」として振る舞い、学習が進むにつれて必要な情報（差分）だけを付加していく、賢い仕組みです。

## 2. MLP (Multi-Layer Perceptron) の役割

Attention が「単語間のやり取り（横の繋がり）」なら、MLP は「単語単体の意味の深掘り（縦の深さ）」です。

GPT-2 の MLP は、各トークンを一度 768次元から **3072次元（4倍）** に拡張し、**GELU 活性化関数** という「窓」を通して情報をフィルタリングし、再び 768次元に戻します。
この高次元空間への投影こそが、複雑なニュアンスや知識を蓄積する源泉となっています。

---

### 実際の実行結果（例）

実際に `uv run experiments/04_block.py` を実行すると、以下のような結果が得られます。

入力文: `Machine Learning` (2トークン)

```text
Step 1: Attention (文脈の統合)
  入力ベクトルの標準偏差: 0.3131
  Attention 出力の標準偏差: 1.1768
  残差接続後の標準偏差: 1.2521

Step 2: MLP (特徴の抽出と深掘り)
  MLP 出力の標準偏差: 3.4770
  最終出力の標準偏差: 3.9117
```

### 実行結果の観察ポイント

1. **標準偏差の増大**:
   - Attention (Step 1) や MLP (Step 2) を通過するたびに、ベクトルの標準偏差が大きくなっています。これは、モデルが「単語の基本座標」に対して、文脈や複雑な特徴といった「追加の情報」を書き込んでいっていることを示しています。

2. **残差接続による安定性**:
   - `x = x + output` という形にすることで、各層は「ゼロからベクトルを作る」のではなく、「前の状態に対する差分（アップデート）」を学習すればよくなります。この積み重ねが、深いモデルでも学習を可能にしています。

## 次のステップへのヒント
これで、GPT-2 の核となるパーツはすべて揃いました！

1. **Embedding**: 言語を「座標（ベクトル）」にする。
2. **Contextualization**: 文脈を吸い込み、座標を動的に変える。
3. **Attention**: 文脈を吸い込むための検索システム。
4. **MLP**: 取り込んだ情報を「深掘り」する。
5. **Block**: それらを積み重ねて深い理解を得る。

いよいよ最終ステップです。これらのパーツをすべて組み合わせて、モデルに「次の単語」を予測させ、実際にテキストを生成させてみましょう。

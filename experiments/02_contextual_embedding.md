# 文脈付き埋め込み (Contextual Embedding)

ユーザーの洞察は非常に鋭いです。
「埋め込み（Embedding）」とは、単に ID をベクトルに変換すること（WTE）だけではありません。

**文脈付き埋め込み**は、Transformer ブロックのすべての層を通過し、周囲の単語の情報を吸い込んだ後のベクトルのことです。

---

## 1. 静的 (Static) vs 文脈付き (Contextualized)

- **静的な埋め込み (WTE)**:
  - 単語 ID ごとに固定されたベクトル。
  - 文脈に関わらず、"bank" は常に同じ座標です。
- **文脈付き埋め込み (Contextualized)**:
  - 12 層の Transformer Block を通過した後のベクトル。
  - 周囲に "river" があれば「土手」の意味に、"country" や "money" があれば「銀行」の意味に、座標が動的に移動します。

## 2. RAG や文章検索での活用

現代の文章検索（セマンティック検索）や RAG で使われる「文章ベクトル」は、まさにこの Transformer の出力（またはそれを平均したもの）です。

GPT-2 のようなモデルでは、**最後のトークンの位置にあるベクトル**が、それより前のすべての単語を Attention で「見ている」ため、文章全体の意味を要約した強力な表現になっています。

---

## 体験してみよう

このディレクトリにある `05_contextual_embedding.py` を実行すると、文末（最後のトークン）のベクトルを比較することで、「文章全体の意味」がどのように数値化されるかを確認できます。

### 実行方法
```bash
uv run experiments/02_contextual_embedding.py
```

### 実際の実行結果（例）

```text
[文章ベクトル（最後のトークン）の類似度比較]
  文 A <-> 文 B (bank) : 0.9995  (文の構造がほぼ同じ)
  文 A <-> 文 C (全然違う) : 0.9969  (内容が全く異なる)
```

### 実行結果の観察ポイント

1. **文章ベクトルの取得**:
   - GPT-2 のようなモデルでは、最後のトークンのベクトルが、それまでのすべての単語の情報を Attention で吸い込んでいます。これが「文章全体の意味」を要約したベクトルになります。

2. **数値が非常に高い理由**:
   - `0.99xx` という非常に高い数値が並ぶのは、GPT-2 のベクトル空間において「文章の構造（冠詞、名詞、前置詞の並び）」が非常に強い類似性を生んでいるためです。
   - `0.9995` (文A-B) と `0.9969` (文A-C) の**わずかな差**が、モデルが捉えた「意味の違い（river vs country）」を表しています。

3. **専用の埋め込みモデルとの違い**:
   - 今回のような素の GPT-2 モデルは「次の単語の予測」に特化しています。
   - 一方、RAG などで使われる OpenAI の `text-embedding-3-small` や BERT などの専用モデルは、**「意味が似た文章をより近づけ、異なる文章をより遠ざける」** ように追加の学習（対照学習）が行われているため、もっとはっきりとした類似度の差（0.8 と 0.2 など）が現れます。

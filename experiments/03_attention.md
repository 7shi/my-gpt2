# Attention (注意) 機構：文脈の魔法

埋め込み体験では、単語が「座標」であることを学びました。しかし、静的な座標だけでは、言葉の多様な意味（文脈）を表現しきれません。

**Attention** は、周囲の単語の情報を賢く「集めてくる」ことで、自分のベクトルを動的にアップデートする仕組みです。

---

## 1. Q, K, V (Query, Key, Value)

各単語（ベクトル）は、Attention 層を通る際に 3 つの役割を与えられます。

- **Query (Q)**: 「自分は今、何を探しているか？」
- **Key (K)**: 「自分はどんな情報を持っているか？」
- **Value (V)**: 「実際の情報は何か？」

各単語の Q と、他の全単語の K を比較（ドット積を計算）することで、どの単語が自分にとって重要かを数値化します。これが **Attention Score** です。

## 2. スケーリングと因果マスキング

### スケーリング ($\sqrt{d_k}$)
ベクトルの次元が大きくなると、ドット積の値が極端に大きくなり、学習が不安定になります。これを防ぐために、次元数 $\sqrt{d_k}$ で割ることで数値を適正な範囲に保ちます。

### 因果マスキング (Causal Masking)
GPT-2 は「次の単語を予測する」モデルです。推論時に、未来の単語（まだ生成されていない単語）を見てはいけません。
そのため、Attention スコアを計算する際、未来のトークンへの注目度を強制的に `-inf`（マイナス無限大）に設定し、実質的に見えなくします。これを**因果マスキング**と呼びます。

## 3. 文脈の統合

計算された重みに従って、他の単語の **Value (V)** を足し合わせます。これにより、単なる「単語の座標」だったものが、「周囲の文脈を取り込んだ、リッチな意味を持つベクトル」へと変化します。

---

## 体験してみよう

このディレクトリにある `main.py` を実行すると、GPT-2 の第 1 層（最初の Transformer ブロック）がどのように単語に注目しているかを確認できます。

### 実行方法
```bash
uv run experiments/03_attention.py
```

### 実際の実行結果（例）

実際に `uv run experiments/03_attention.py` を実行すると、以下のような結果が得られます。

入力文: `The quick brown fox jumps over the lazy dog.`

1. **注目先の偏り (Head 0)**:
   最後のトークン `.` がどの単語に注目しているかを見ると、次のような傾向が現れます。
   ```text
   最後のトークン '.' が注目している先:
     ->  lazy      : 0.2894
     -> The        : 0.2158
     ->  fox       : 0.1567
     ->  jumps     : 0.1041
   ```
   このように、モデルは文末のピリオドを理解する上で、文の始まり (`The`) や主体 (`fox`)、直前の形容詞 (`lazy`) など、**文法的に重要な単語**に優先的に注目していることが分かります。

2. **因果マスキングの形**:
   最初のトークン `The` の注目先を見ると、結果は次のようになります。
   ```text
   最初のトークン 'The' が注目している先:
     -> The        : 1.0000
   ```
   最初の単語は自分より後の単語（未来の情報）をまだ知らないため、自分自身に 100% 注目するしかありません。これが**因果マスキング**による制約であり、モデルが順番に言葉を生成するために不可欠な仕組みです。

### 次のステップへのヒント
この Attention によって、各トークンは単なる「自分の座標」から、「周囲の文脈を取り込んだリッチなベクトル」へとアップデートされました。

しかし、文脈を取り込むだけでは不十分です。取り込んだ情報をさらに「深掘り」し、より高度な意味を抽出する必要があります。その役割を担うのが、次のステップである **MLP（多層パーセプトロン）** です。

Attention（文脈の結合）と MLP（情報の深掘り）をセットにした **Transformer Block** の全体像を、**`experiments/04_block.md`** で見ていきましょう。

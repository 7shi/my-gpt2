# テキスト生成 (Generation)：創造性の仕組み

いよいよ最後のステップです。これまでの「埋め込み」「Attention」「MLP」のすべての計算結果が、最終的に「次の 1 単語（トークン）の出現確率」として出力されます。

この確率分布からどのように次の単語を「選ぶか」によって、モデルの振る舞いが大きく変わります。

---

## 1. 確率分布から単語を選ぶ

GPT-2 の出力層は、語彙（約5万個）のすべてに対して「次に現れる確率」を算出します。

### サンプリング手法（選び方）

1. **Greedy Search (貪欲法)**:
   常に最も確率が高い単語を 1 つだけ選びます。
   - **特徴**: 一貫性がありますが、文章が長くなると同じフレーズを繰り返す「ループ」が発生しやすくなります。
2. **Top-K サンプリング**:
   確率の高い上位 $k$ 個の単語の中から、ランダムに（重み付けをして）選びます。
   - **特徴**: 意外性や創造性が加わり、人間らしい文章になります。
3. **Top-P (Nucleus) サンプリング**:
   累積確率が $p$（例：0.9）に達するまでの単語の中から、ランダムに選びます。
   - **特徴**: 語彙の分布に応じて、候補の数を柔軟に変えることができます。

## 2. Temperature (温度)

確率分布の「尖り具合」を調整するパラメータです。

- **低い温度 (T < 1.0)**: 確率の高い単語がより選ばれやすくなり、**保守的**な生成になります。
- **高い温度 (T > 1.0)**: 確率の低い単語にもチャンスが与えられ、**多様で創造的**な生成になります。

## 3. 自己回帰 (Auto-regressive)

GPT-2 は一度に長い文章を作ることはできません。

1. プロンプト（入力）を受け取り、次の 1 単語を予測する。
2. その単語をプロンプトに**付け加える**。
3. 新しいプロンプトを使って、さらに次の 1 単語を予測する。

このループを繰り返すことで、一歩ずつ文章を紡ぎ出していきます。これを「自己回帰」と呼びます。

## 体験してみよう

このディレクトリにある `04_generation.py` を実行すると、モデルが次の単語を予測する際の「上位候補」とその確率をステップごとに確認できます。

### 実行方法
```bash
uv run experiments/05_generation.py
```

### 実際の生成プロセス（例）

実際に `uv run experiments/05_generation.py` を実行すると、モデルが次のように言葉を選んでいることが分かります。

プロンプト: `Artificial Intelligence will`

1. **Step 1: 迷いのある選択**
   上位候補には `be` (0.1837), `become` (0.0355), `not` (0.0351) などが挙がります。どれが選ばれても自然ですが、確率が分散しているため、ここでの選択がその後の文章の方向性を決めます。

2. **Step 3: 確信を持った選択**
   `Artificial Intelligence will be able` まで生成されると、次のトークンは `to` が **0.9907** という圧倒的な確率で選ばれます。熟語や文法的なパターンが確定すると、モデルの「迷い」は消え、決定的（デターミニスティック）な振る舞いになります。

## 旅の総括：GPT-2 の全体像

おめでとうございます！これで GPT-2 の仕組みをひと通り巡りました。

1. **01_embedding**: 言語を「座標（ベクトル）」に変換する
2. **02_contextual_embedding**: 文脈を吸い込み、座標を動的に変化させる
3. **03_attention**: 情報を吸い込むための「検索（Q, K, V）」の仕組み
4. **04_block**: それを積み重ね、高次元で情報を「深掘り（MLP）」する
5. **05_generation**: 最後に、最も自然な単語を「確率」で選び出す

この一連の流れが、現代の驚異的な AI たちの根底にある基本原則です。

この基礎知識があれば、プロジェクトのソースコード（`my_gpt2/model.py` など）を読んだり、自分自身でモデルを「学習」させたりする準備は万端です。

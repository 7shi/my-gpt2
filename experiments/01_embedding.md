# 埋め込み (Embedding) と RAG の基礎

GPT-2 の内部構造を詳しく見ていく前に、この「埋め込み（Embedding）」という概念を理解することは非常に重要です。
なぜなら、埋め込みこそが「人間の言葉」と「AIの計算空間」を結びつける唯一の橋渡しだからです。

---

## 1. 埋め込み (Embedding) とは？

コンピュータは「単語」をそのまま理解することはできません。
「リンゴ」と「バナナ」はコンピュータにとっては単なる異なる記号ですが、人間にとっては「どちらも果物である」という共通点があります。

**埋め込み**は、単語（トークン）を多次元（GPT-2 small では 768次元）の**ベクトル（数値のリスト）**に変換することで、この「意味の近さ」を座標上の距離として表現します。

- 意味が似ている単語（例：cat と dog）は、ベクトル空間上で近くに配置されます。
- 意味が異なる単語（例：cat と apple）は、遠くに配置されます。

このプロジェクトの `my_gpt2/model.py` にある `WTE` (Word Token Embedding) 行列が、まさにこの「単語を座標に変換する辞書」の役割を果たしています。

## 2. コサイン類似度：ベクトルの「向き」を比べる

2つの単語がどの程度似ているかを測る最も一般的な指標が**コサイン類似度**です。
これは、2つのベクトルがなす角度のコサイン（$cos \theta$）を計算します。

- **1.0**: 全く同じ向き（完全に一致）
- **0.0**: 直交している（無関係）
- **-1.0**: 真逆の向き

`embedding/main.py` では、この計算を使って単語の類似度を求めています。

## 3. ベクトル演算：意味の引き算・足し算

面白いことに、埋め込み空間では意味の演算が可能になることがあります。
有名な例は **「王 - 男 + 女 = 女王」** ですが、実は GPT-2 のようなモデルの「静的な埋め込み（WTE）」だけでは、この計算結果が常に綺麗に「女王」になるとは限りません。

GPT-2 は、この静的なベクトルを **Attention（注目）機構** によって「文脈」に合わせて変化させることで、初めて高度な意味の操作を実現しています。

## 体験してみよう

このディレクトリにある `main.py` を実行すると、GPT-2 が持っている実際の「意味空間」を覗き見ることができます。

### 実行方法
```bash
uv run experiments/01_embedding.py
```

### 実際の実行結果（例）

実際に `uv run embedding/main.py` を実行すると、以下のような結果が得られます。

1. **類似度の比較**:
   ```text
   cat      <-> dog      : 0.3815
   cat      <-> apple    : 0.2480  (動物同士の方が類似度が高い)
   man      <-> woman    : 0.5863  (非常によく似た文脈で使われる)
   ```

2. **最近傍探索 (Nearest Neighbor)**:
   `king` に近いトークンを探すと、大文字小文字の違いや、BPE による断片（`ker`, `kers` など）が現れます。
   ```text
   king       (ID:  3364, Sim: 1.0000)
   ked        (ID:  9091, Sim: 0.6833)
   KING       (ID: 37286, Sim: 0.5702)
   King       (ID: 15708, Sim: 0.5314)
   ```

3. **ベクトル演算の結果**:
   `king - man + woman` を計算した結果は以下の通りです。
   ```text
   king       (ID:  3364, Sim: 0.7752)
   woman      (ID:  8580, Sim: 0.5815)
   women      (ID: 25878, Sim: 0.4754)
   ```
   期待した `queen` はトップ 5 に入っていません。代わりに `women` がランクインしており、意味的な「女性性」の方向へベクトルが移動していることは分かります。

### 次のステップへのヒント
この体験で「単語が 768 次元の座標である」という実感が持てれば準備万端です。
しかし、座標だけでは「意味」を十分に扱い切れないことも分かりました（`queen` がズバリ出ないなど）。

GPT-2 の真価は、この静的な座標を **Attention** という仕組みで「周りの単語の情報」と混ぜ合わせ、文脈に即したベクトルへと動的にアップデートしていくところにあります。

次は、その静的な座標を **12層の Transformer** を通して「文脈付きのベクトル」へと動的にアップデートしていく **`experiments/02_contextual_embedding.md`** を覗いてみましょう。

# MLP: トークンの特徴変換

MLP（Multi-Layer Perceptron）は、各トークンを独立に処理する特徴変換です。Attention（👉[06](06_attention.md)）が「トークン間の関係」を扱うのに対し、MLP は「トークン単体の意味の深掘り」を担当します。

## 1. MLP の役割

Attention はトークン間で情報を集約しますが、集めた情報を「解釈」する処理は行いません。MLP がその役割を担います。768 次元のベクトルを一度 4 倍の 3072 次元に拡張し、活性化関数で情報をフィルタリングしてから元の 768 次元に圧縮します。

重みとバイアスは safetensors ファイルから読み込まれます。

```python
MLP(
    w_fc=weights["h.0.mlp.c_fc.weight"],      # (768, 3072) 拡張
    b_fc=weights["h.0.mlp.c_fc.bias"],        # (3072,)
    w_proj=weights["h.0.mlp.c_proj.weight"],  # (3072, 768) 射影
    b_proj=weights["h.0.mlp.c_proj.bias"],    # (768,)
)
```

MLP の処理は 2 つの線形変換と、その間の活性化関数で構成されます。

```python
class MLP:
    def __call__(self, x):
        a = gelu(x @ self.w_fc + self.b_fc)   # 768 → 3072（拡張＋活性化）
        return a @ self.w_proj + self.b_proj  # 3072 → 768（射影）
```

```
入力:   (seq_len, 768)
中間層: (seq_len, 3072)   ← 4倍に拡張 + GELU
出力:   (seq_len, 768)    ← 元に戻す
```

## 2. 拡張: 768 → 3072

最初の線形変換は、768 次元の入力に (768, 3072) の重み行列を掛けて 3072 次元に拡張します。Attention の Q, K, V 生成と同じくアフィン変換（行列積 + バイアス）です。

```python
intermediate = x @ self.w_fc + self.b_fc  # (seq_len, 768) → (seq_len, 3072)
```

入力 "The capital of France is" での ' is' の中間層を見ると、3072 次元のうち正の値は 284 個だけで、大部分（2788 個）が負の値になっています。この偏りが次の GELU で重要な役割を果たします。

### 設計の動機：なぜ 4 倍に拡張するのか？

低次元空間では区別が難しい複雑なパターンも、高次元に引き上げると分離しやすくなります。たとえば 2 次元の平面上では分けられない点の集まりも、3 次元に持ち上げれば平面で切り分けられることがあります。3072 次元への拡張はこの原理の延長です。

## 3. GELU 活性化関数

**GELU** (Gaussian Error Linear Unit) は ReLU の滑らか版です。ReLU が $x < 0$ で完全にゼロにするのに対し、GELU は負の領域でも小さな値を通します。

$$
\text{GELU}(x) \approx 0.5 \times x \times \left(1 + \tanh\left(\sqrt{\frac{2}{\pi}} \times (x + 0.044715 \times x^3)\right)\right)
$$

```python
def gelu(x):
    return 0.5 * x * (1 + np.tanh(np.sqrt(2 / np.pi) * (x + 0.044715 * x ** 3)))
```

代表的な入力値に対する GELU の出力：

| 入力 | GELU 出力 | 備考 |
|-----:|----------:|------|
| -2.0 | -0.0454 | ほぼ抑制 |
| -1.0 | -0.1588 | 小さな負の値を通す |
| -0.5 | -0.1543 | 最小値付近 |
|  0.0 |  0.0000 | 原点 |
| +0.5 |  0.3457 | 正の領域はほぼ線形 |
| +1.0 |  0.8412 | 入力に近い値 |
| +2.0 |  1.9546 | ほぼ恒等写像 |

正の値はほぼそのまま通過し、負の値は大きく抑制されます。ReLU（$\max(0, x)$）との違いは、境界付近で急激にゼロにならず滑らかに遷移する点です。

### GELU によるフィルタリングの効果

実際の中間層での GELU の効果を見てみます。

```
中間層の要素数: 15360（5トークン × 3072次元）
負の入力: 13520 (88.0%)
GELU で抑制（|出力| < 0.01）: 656 (4.3%)
活性化後にほぼゼロ（|出力| < 0.1）: 7419 (48.3%)
```

3072 次元の約半数が GELU によってほぼゼロに抑制されています。トークンごとに見ると、活性化パターンが異なることが分かります。

| トークン | 正の入力 | GELU 後にほぼゼロ |
|----------|----------|--------------------|
| 'The' | 210/3072 | 2037/3072 (66.3%) |
| ' capital' | 419/3072 | 1066/3072 (34.7%) |
| ' of' | 276/3072 | 1670/3072 (54.4%) |
| ' France' | 651/3072 | 1025/3072 (33.4%) |
| ' is' | 284/3072 | 1621/3072 (52.8%) |

' France' は 651 次元が活性化する一方、'The' は 210 次元しか活性化しません。これは各トークンが MLP 内で異なる「特徴パターン」を発火させていることを意味します。GELU は、3072 次元のうちトークンにとって重要な特徴だけを選択的に通過させるフィルタとして機能しています。

' is' の上位活性化次元を見ると、正の入力値はほぼそのまま通過しています。

```
dim  874: 中間層 +3.6473 → GELU +3.6470（ほぼ同値）
dim  774: 中間層 +2.9100 → GELU +2.9052
dim 2845: 中間層 +2.5028 → GELU +2.4878
```

一方、負の入力はほとんど消されます。

```
dim 1: 中間層 -1.9634 → GELU -0.0486（大幅に抑制）
dim 3: 中間層 -1.9311 → GELU -0.0516
dim 4: 中間層 -0.9169 → GELU -0.1648
```

## 4. 射影: 3072 → 768

GELU で選別された 3072 次元のベクトルに (3072, 768) の重み行列を掛けて、元の 768 次元に戻します。

```python
output = activated @ self.w_proj + self.b_proj  # (seq_len, 3072) → (seq_len, 768)
```

この射影は、GELU で活性化された次元の情報を 768 次元に集約する処理です。ゼロに近い次元は結果にほとんど寄与しないため、実質的に活性化された次元だけが出力に反映されます。

## 5. MLP によるベクトルの変化

MLP 適用前後のベクトルを比較すると、大きな変化が起きていることが分かります。

```
      トークン   入力std   出力std   コサイン類似度
       The    0.2058    4.4876      0.3055
   capital    0.1636    1.0964      0.3250
        of    0.1539    1.0827      0.2095
    France    0.1494    1.2973      0.3210
        is    0.1472    1.1864      0.1841
```

コサイン類似度が 0.2〜0.3 と低く、MLP がベクトルの方向を大きく変えていることが分かります。標準偏差も大幅に増加しており、MLP が情報を積極的に書き込んでいます。特に 'The' は標準偏差が 0.21 から 4.49 と約 22 倍に増加しています。

ただし、この出力はそのまま次の層に渡されるのではなく、残差接続（👉[08](08_residual.md)）によって元のベクトルに加算されます。MLP の大きな変化も、残差接続で元のベクトルに足し合わされることで緩和され、情報の上書きではなく「追記」として機能します。

### 補足：Attention と MLP の分業

Transformer Block 内の Attention と MLP は異なる役割を担っています。

- **Attention**: トークン間の情報伝達。「どの単語を参照するか」を決めて情報を集める（横方向の処理）
- **MLP**: トークン単体の特徴変換。集めた情報を解釈し、次の層に向けて意味を加工する（縦方向の処理）

この 2 つが交互に繰り返されることで、トークンのベクトルは徐々に「次の単語を予測する」ために必要な表現へと変化していきます。

## まとめ

GPT-2 の MLP の処理は以下の流れで進みます。

1. 768 次元のベクトルに重み行列を掛けて 3072 次元に拡張する
2. GELU 活性化関数でトークンごとに重要な特徴を選別する
3. 3072 次元のベクトルに重み行列を掛けて 768 次元に射影する

構造は単純ですが、3072 次元への拡張と GELU による選別により、各トークンに固有の特徴パターンを書き込む重要な役割を果たしています。

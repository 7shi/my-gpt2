# はじめに

LLM（大規模言語モデル）の内部構造に関する書籍は増えていますが、本を読んだだけでは「分かった気になる」で終わりがちです。結局、手を動かさないと理解は難しいのですが、「どうやって手を動かせば良い」のでしょうか。

LLM の自作資料の多くは、モデルの学習から入ります。しかし学習は推論より難しく、データパイプライン・損失関数・最適化・勾配計算をひととおり理解してからでないとモデルが動きません。覚えることが多く、動くものを見る前に力尽きてしまいがちです。

このプロジェクトでは敢えて学習要素を省き、既存の GPT-2 の重みを使った推論に特化しています。最初から意味のある出力が得られるため、「動くものを見てから仕組みを深掘りする」という流れで学習を進められます。

## 構成

GPT-2 の推論パイプラインを、データの流れに沿って分解します。

テキストはまず**トークナイザー**によって整数列に変換されます。英語モデルではバイトレベル BPE、日本語モデルでは SentencePiece（ユニグラムモデル）を使います。次に **Embedding** がこの整数列を連続ベクトル空間に写像し、**Transformer Block**（LayerNorm → Attention → 残差接続 → LayerNorm → MLP → 残差接続）を 12 回繰り返すことでベクトルを変換します。最後に **LM Head** がベクトルを語彙全体の確率分布に変換し、**サンプリング**によって次のトークンが選ばれます。

各章では、この流れの一区間を取り出し、NumPy で動くコンパクトな推論エンジンを動かしながら、「なぜその計算をするのか」を確認していきます。
